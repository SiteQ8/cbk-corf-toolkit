# Contributing to CBK CORF Assessment Toolkit

Thank you for your interest in contributing to the CBK CORF Assessment Toolkit! This project aims to support Kuwait's financial sector in understanding and achieving compliance with the Central Bank of Kuwait's Cyber and Operational Resilience Framework.

## 🤝 How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest enhancements
- Provide detailed information about the issue
- Include screenshots if applicable
- Specify browser version and environment

### Suggesting Enhancements
- Open an issue with the "enhancement" label
- Clearly describe the proposed feature
- Explain the benefits to users
- Consider implementation complexity

### Code Contributions

#### 1. Fork the Repository
```bash
git clone https://github.com/siteq8/cbk-corf-toolkit.git
cd cbk-corf-toolkit
```

#### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

#### 3. Make Your Changes
- Follow existing code style and structure
- Maintain bilingual support (English/Arabic)
- Test in multiple browsers
- Ensure responsive design works

#### 4. Test Thoroughly
- Test language switching functionality
- Verify all assessment calculations
- Check mobile responsiveness
- Validate HTML/CSS/JS

#### 5. Commit Changes
```bash
git add .
git commit -m "Add: Brief description of changes"
```

Use conventional commit messages:
- `Add:` New features
- `Fix:` Bug fixes
- `Update:` Updates to existing features
- `Refactor:` Code refactoring
- `Docs:` Documentation changes

#### 6. Push and Create Pull Request
```bash
git push origin feature/your-feature-name
```

Then create a Pull Request on GitHub with:
- Clear description of changes
- Reference to related issues
- Screenshots if UI changes

## 📋 Contribution Guidelines

### Code Style
- Use consistent indentation (4 spaces)
- Comment complex logic
- Use descriptive variable names
- Keep functions focused and small

### Bilingual Requirements
- All UI text must support English and Arabic
- Update both language objects in translations
- Test RTL layout for Arabic
- Ensure proper text direction handling

### Design Principles
- Maintain existing color scheme and design system
- Keep responsive design principles
- Ensure accessibility standards
- Use CSS variables for theming

### Testing Requirements
- Test in Chrome, Firefox, Safari, and Edge
- Verify mobile responsiveness
- Test both languages thoroughly
- Validate assessment calculations

## 🎯 Priority Areas for Contribution

### High Priority
1. **Additional Assessment Questions**: Expand questionnaire depth
2. **Gap Analysis Templates**: Create downloadable templates
3. **Control Mapping**: Map CORF to other frameworks (ISO, NIST)
4. **Export Functionality**: PDF export of results
5. **Implementation Guides**: Step-by-step compliance guides

### Medium Priority
1. **Enhanced Visualizations**: Charts and graphs for results
2. **Progress Tracking**: Save and compare assessments over time
3. **Maturity Roadmaps**: Visual maturity progression paths
4. **Best Practices Library**: Curated implementation resources
5. **Training Materials**: Educational content and guides

### Low Priority
1. **Advanced Analytics**: Trend analysis and benchmarking
2. **Multi-user Support**: Collaborative assessment features
3. **API Integration**: Connect with other tools
4. **Mobile App**: Native mobile applications
5. **AI Recommendations**: ML-powered guidance

## 🔍 Review Process

1. **Initial Review**: Maintainer reviews PR within 3-5 days
2. **Feedback**: Comments and requested changes if needed
3. **Testing**: Maintainer tests changes thoroughly
4. **Approval**: PR approved if meets all requirements
5. **Merge**: Changes merged into main branch

## 🌍 Translation Contributions

We welcome translation improvements!

### Arabic Translation
- Ensure proper terminology for financial/security terms
- Maintain professional tone
- Verify RTL layout compatibility
- Use appropriate Arabic fonts

### Additional Languages
- Open an issue to propose new language support
- Provide complete translations
- Test thoroughly with RTL/LTR considerations

## 🚫 What Not to Contribute

Please avoid:
- Changes that contradict CBK official guidance
- Unofficial interpretations of CORF requirements
- Features that require backend servers
- Breaking changes to existing functionality
- Removal of educational disclaimers
- Misrepresentation as official CBK tool
